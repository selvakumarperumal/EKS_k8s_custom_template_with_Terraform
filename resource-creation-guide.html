📋 Quick Reference — All Steps at a Glance

🌐 Phase 1 — VPC Module

# Step Description
1 Create VPC Isolated network (10.0.0.0/16) with DNS support enabled — required for EKS service discovery and API endpoint resolution.
2 Internet Gateway Single entry point for public internet traffic. Attached to VPC, used by public subnets only.
3 Public Subnets (×3) One per AZ for ALB and NAT Gateway placement. Auto-assigns public IPs. Tagged for K8s external load balancers.
4 Private Subnets (×3) One per AZ for EKS worker nodes. No public IPs — all outbound traffic goes through NAT. Tagged for K8s internal LBs.
5 NAT Gateway + EIP Allows private nodes to pull images and make API calls. Single (dev, ~$33/mo) or per-AZ (prod, ~$99/mo) for HA.
6 Route Tables Public RT routes 0.0.0.0/0 → IGW. Private RTs route 0.0.0.0/0 → NAT. Per-AZ private RTs for fault isolation in prod.
7 Network ACLs Stateless subnet-level firewall. Public allows 80/443/ephemeral. Private allows only VPC-internal traffic. Second defense layer.
8 VPC Flow Logs Optional. Captures all network traffic metadata to CloudWatch. Required for PCI DSS / HIPAA compliance. ~$5/month.

🔐 Phase 2 — IAM Module

# Step Description
9 Cluster IAM Role Trusted by eks.amazonaws.com. Grants AmazonEKSClusterPolicy (manage cluster) + VPCResourceController (create ENIs for pods).
10 Node Group IAM Role Trusted by ec2.amazonaws.com. 3 policies: WorkerNodePolicy (join cluster), CNI Policy (pod networking), ECR ReadOnly (pull images).

⬡ Phase 3 — EKS Module

# Step Description
11 KMS Key Envelope encryption for K8s secrets in etcd. Auto-rotates annually. Without this, secrets are only base64-encoded.
12 Security Groups Cluster SG + Node SG with 3 rules: Node→Cluster (443), Cluster→Node (1025-65535), Node↔Node (all ports for pod traffic).
13 EKS Cluster The control plane (~8-10 min). 3 API server replicas across 3 AZs. Dual access (public + private). KMS-encrypted secrets. 5 log types.
14 OIDC Provider (IRSA) Bridges K8s ServiceAccount identity → AWS IAM roles. Allows per-pod IAM permissions instead of shared node-level access.
15 EKS Addons CoreDNS (service discovery), kube-proxy (Service routing via iptables), VPC CNI (assigns real VPC IPs to pods).
16 Launch Templates EC2 config per node group: gp3 encrypted EBS (3000 IOPS), IMDSv2 enforced (blocks SSRF), no public IPs, detailed monitoring optional.
17 Managed Node Groups Actual EC2 workers (~3-5 min). AZ-balanced spread. Supports ON_DEMAND + SPOT. lifecycle ignores desired_size for Cluster Autoscaler.

🔑 Phase 4 — Secrets Manager Module

# Step Description
18 Dedicated KMS Key Separate from EKS KMS key — key isolation so one compromise doesn't expose both cluster secrets and app secrets.
19 Create Secrets Conditionally creates DB credentials, API keys, and app config in Secrets Manager. Values flow from GitHub Secrets → TF_VAR_* → SM.
20 IAM Read Policy Least-privilege policy: GetSecretValue + KMS Decrypt for only the created secrets. Attach to IRSA roles for pod access.

🛡️ Phase 5 — Security Module

# Step Description
21 GuardDuty Detector Account-level ML threat detection. Auto-discovers all EKS clusters, VPCs, EC2s — no ARN needed. Detects crypto mining, credential theft, privilege escalation.
22 AWS Config + Rules Continuous configuration compliance. 3 EKS rules: Is logging enabled? Is endpoint private? Are secrets KMS-encrypted? Flags but doesn't auto-remediate.
