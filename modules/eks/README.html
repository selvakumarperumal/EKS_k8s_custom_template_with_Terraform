⬡ Module Overview — What Gets Created

This is the largest and most critical module (~760 lines). It provisions the entire EKS cluster with all dependencies in the correct order.

🔑 KMS Key→ 📊 CloudWatch→ 🛡️ Security Groups→ ⬡ EKS Cluster (~10 min)→ 🔐 OIDC→ 📦 Add-ons→ 🖥️ Nodes
  • KMS Key: Envelope encryption for Kubernetes secrets in etcd — annual auto-rotation
  • CloudWatch Logs: Optional control plane logging (api, audit, authenticator, controllerManager, scheduler)
  • Security Groups: Cluster SG + Node SG with 3 inter-component rules (minimal blast radius)
  • EKS Cluster: Managed control plane — 3 redundant API servers across 3 AZs
  • OIDC Provider: Enables IRSA — per-pod IAM via ServiceAccounts (eliminates shared node role)
  • Add-ons: VPC CNI (real VPC IPs for pods), kube-proxy (iptables routing), CoreDNS (service discovery)
  • Launch Templates: IMDSv2 enforced, gp3 encrypted EBS (3000 IOPS), no public IPs, detailed monitoring
  • Node Groups: ON_DEMAND (reliable) + SPOT (cost savings) with autoscaler-safe lifecycle rules
📂 Source: modules/eks/main.tf · variables.tf · outputs.tf · Total deploy time: ~17 min
